# PRD 31 — Rollout & Enablement

> **Source:** PA 31, lines 12949–13136.

## Area overview

Two flavors of rollout: **internal** (the team building ConqrAI Wiki adopts ConqrAI Wiki) and **customer-facing** (training and enablement programs).

## Epic 31.1 — Internal Rollout

### Feature 31.1.1 — Dogfooding plan

The product team uses ConqrAI Wiki as their primary documentation system from week 1. Surfaces real friction faster than any QA suite.

**Practices.**
- Weekly "wiki health" review
- Bug filed for any documentation friction
- Feature requests prioritized when the team itself feels the pain
- Onboarding new hires through ConqrAI Wiki

### Feature 31.1.2 — Departmental adoption ladder

1. Engineering → owns the platform; first adopter
2. Product + Design → second wave; uses the spec / design templates
3. Sales / CS → third wave; client docs and runbooks
4. HR / Operations → fourth wave; policy docs
5. Legal / Finance → fifth wave; regulated content

## Epic 31.2 — Customer Onboarding

### Feature 31.2.1 — Onboarding flow

- Workspace setup wizard
- Sample data templates (so empty workspaces feel useful)
- Quickstart checklist (invite team, create first space, first page, install MCP, …)

### Feature 31.2.2 — Migration assistance

- Notion / Confluence / Google Docs import wizard
- Concierge migration for Enterprise (assisted by customer success)
- Mapping IdP groups → workspace groups

## Epic 31.3 — Customer Training & Enablement

### Feature 31.3.1 — Documentation

Use ConqrAI Wiki itself for customer-facing documentation (eat your own).

### Feature 31.3.2 — Training programs

- Self-serve video walkthroughs
- Live onboarding sessions for Enterprise
- Admin certification (planned)
- Best-practice library

### Feature 31.3.3 — Community

- Discord / Slack community
- Public roadmap (read-only Notion-style portal — built on ConqrAI Wiki itself)
- Quarterly webinars on documentation governance

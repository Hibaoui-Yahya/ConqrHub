# PRD 29 — Advanced AI & Automation

> **Source:** PA 29, lines 11797–12101.

## Area overview

Long-horizon AI capabilities that build on the four shipped surfaces (Generative AI, AI Search, AI Chat, MCP). Most are planned; this PRD captures intent and architecture pre-design.

## Epic 29.1 — Knowledge Graph & Semantic Intelligence

### Feature 29.1.1 — Entity extraction

Pull entities (people, teams, products, systems, customers) from page content. Build a workspace-wide knowledge graph that AI Chat can traverse.

### Feature 29.1.2 — Relationship inference

"X depends on Y", "A is owned by team B", "C is a runbook for D" — relationships inferred from content + explicit annotations.

### Feature 29.1.3 — Graph-aware AI answers

When a user asks "Who owns the payments service?", AI Chat traverses the graph rather than re-extracting from raw text every time.

## Epic 29.2 — Documentation Workflow Automation

### Feature 29.2.1 — Auto-generate docs from code

A repo connector watches commits / PRs and auto-creates / updates pages: API reference from OpenAPI, runbooks from CI signals, release notes from PR descriptions.

### Feature 29.2.2 — Auto-update outdated content

When a referenced API changes, the AI proposes an edit to the page that documents it. Author reviews and accepts.

### Feature 29.2.3 — Auto-merge duplicates

Semantic duplicate detection + AI-proposed merge with conflict resolution UI.

### Feature 29.2.4 — Auto-archive obsolete

Pages with no views, no recent edits, and superseded by newer content are flagged for archival.

## Epic 29.3 — Intelligent Onboarding Paths

### Feature 29.3.1 — Role-based reading lists

A new engineer sees a curated path through the wiki: setup → repo overview → on-call basics → first deploy. Path defined by the role's space admin or generated by AI from common patterns.

### Feature 29.3.2 — Progress tracking

User checks off pages; manager sees onboarding progress.

### Feature 29.3.3 — Guided AI tutor

The AI assistant walks new hires through the path, answering questions in context.

## Epic 29.4 — Role-Based AI Assistants

### Feature 29.4.1 — Persona-tuned prompts

Engineer-mode (technical depth, code examples) · Sales-mode (concise, customer-friendly) · Support-mode (verified-source bias). Each persona has tailored prompts and tool access scopes.

### Feature 29.4.2 — Tool access scopes

A "support agent" persona may have read-only tool access; an "engineer" persona may have create_page / update_page; an admin persona may have create_space.

## Status

All Epic 29 items are **roadmap**. The four shipped AI surfaces (Generative AI, AI Search, AI Chat, MCP) provide the substrate.
